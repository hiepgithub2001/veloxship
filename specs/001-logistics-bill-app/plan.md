# Implementation Plan: Ứng dụng Quản lý Phiếu Gửi Vận Chuyển

**Branch**: `001-logistics-bill-app` | **Date**: 2026-04-26 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-logistics-bill-app/spec.md`

## Summary

Build a web application for a Vietnamese express-delivery carrier whose primary v1 capability is creating, looking up, reprinting, and status-tracking "Phiếu Gửi" (delivery bills) that visually match the supplied paper template. The system also stores reusable customer profiles to auto-fill the sender block and uses authenticated staff accounts so every action is auditable.

Per user direction the technical approach is:

- **Frontend**: React (JavaScript)
- **Backend**: Python with FastAPI
- **Database & migrations**: PostgreSQL with SQLAlchemy
- **Authentication**: JWT-based login

The entire end-user surface is in Vietnamese (UI labels, validation messages, printed output, dates), and search must be diacritic-insensitive.

## Technical Context

**Language/Version**: Python 3.11+ (backend); JavaScript ES2022 (frontend) on Node.js 20 LTS
**Primary Dependencies**:

- Backend: FastAPI, Uvicorn, SQLAlchemy 2.x (async), Alembic, Pydantic v2, `python-jose[cryptography]` (JWT), `passlib[bcrypt]` (password hashing), `python-multipart`, `qrcode`, `python-barcode`, `weasyprint` (HTML→PDF for printed bill), `unidecode` (diacritic-insensitive helpers).
- Frontend: React 18, Vite (dev/build), React Router, Axios, React Query (server state), React Hook Form + Zod (forms/validation), Ant Design (Vietnamese-friendly admin UI with built-in `vi_VN` locale), `dayjs` with `vi` locale, `jsbarcode`, `qrcode.react`, `react-to-print` (or print CSS).

**Storage**: PostgreSQL 16 with `unaccent` and `citext` extensions enabled; SQLAlchemy 2.x async ORM; Alembic migrations. Audit trail stored in a dedicated `audit_event` table.
**Testing**: Backend — `pytest`, `pytest-asyncio`, `httpx` (AsyncClient), `pytest-postgresql` or testcontainers for integration tests. Frontend — `vitest`, `@testing-library/react`, `msw` for API mocking, Playwright for end-to-end smoke (bill creation + print preview).
**Target Platform**: Web application served from a Linux server. Browser support: latest two stable versions of Chrome / Edge / Firefox / Safari, plus reasonable rendering on tablet form factors used at counters.
**Project Type**: Web application (frontend + backend). Layout follows the standard `backend/` + `frontend/` split.
**Performance Goals**: Bill list / search returns ≤ 1 s for 100k rows (SC-004); reprint and status update ≤ 2 s end-to-end (SC-009); bill creation API latency p95 ≤ 300 ms server-side; full bill creation flow ≤ 90 s for a returning customer (SC-001).
**Constraints**:

- Vietnamese-only UI; full diacritic preservation; diacritic-insensitive search (PostgreSQL `unaccent`).
- Vietnamese decimal format (comma) accepted for input and rendered consistently.
- Tracking numbers must be unique under concurrency (DB unique constraint + sequence-based generation, SC-006).
- Printed bill must be visually faithful to `image_data/bill_format.jpg` (SC-003) on standard office paper.
- Online operation only (offline-first explicitly out of scope per spec).
- Single-tenant (one carrier) for v1.

**Scale/Scope**: Order-of-magnitude target: ~10–50 counter staff users, up to ~100k bills/year per carrier, ~5k saved customer profiles. Single PostgreSQL primary; horizontal scaling not required for v1.

**Structure Decision**: Web application, two top-level deliverables (`backend/` and `frontend/`). The backend follows the FastAPI-recommended layered layout (api → services → crud → models). The frontend is a feature-folder Vite + React app with a single Vietnamese string catalogue (`src/i18n/vi.js`) used via a translation helper, ensuring the UI is exclusively in Vietnamese (FR-021). PostgreSQL with the `unaccent` extension powers diacritic-insensitive search (FR-014). A WeasyPrint-rendered HTML template under `backend/app/static/bill_template.html` produces the print-exact PDF (FR-009, FR-012); the frontend uses the same template rendered in-browser for on-screen preview and physical printing.
